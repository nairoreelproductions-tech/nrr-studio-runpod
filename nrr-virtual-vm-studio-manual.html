01 — Overview

How the system works

NRR Studio runs on a central VPS that holds all shared project files, Blender builds, and asset libraries. Each team member connects to it through a cloud workstation (VNC desktop) that is bootstrapped automatically. Files sync between your local workspace and the VPS using rclone over a private SSH connection.

Key concept Your ~/studio/ folder is your local workspace. The VPS is the shared source of truth. Syncing is not automatic on pull — you control when you fetch new work.

Auto-sync

Your PROJECTS folder pushes to the VPS automatically every 5 minutes in the background. You don't need to do anything for your saves to reach the server — but you do need to manually pull to receive other people's changes.

02 — Setup

First-time setup

You only need to run this once per machine or new VNC session. Ask the team lead for your VPS_SSH_KEY_B64 value before starting.

  1. Open a terminal inside the VNC desktop.
  2. Export your SSH key (provided by the team lead — keep this private): 
    export VPS_SSH_KEY_B64="paste_your_key_here"
  3. Run the bootstrap script: 
    curl -fsSL https://raw.githubusercontent.com/YOUR_REPO/bootstrap.sh | bash
  4. Wait for it to finish. It will download all project files, set up Blender, and register all aliases. This can take several minutes depending on project size.
  5. Reload your shell so aliases are available: 
    source ~/.bashrc
  6. Launch Blender from the Desktop shortcut or type studio-open in the terminal.
Important Never share your VPS_SSH_KEY_B64 in chat, email, or commits. Treat it like a password.
03 — Folder Structure

What lives where

~/studio/
├── PROJECTS/ ← your working files (synced every 5 min)
│ ├── my_scene/
│ └── client_render/
├── LIBRARY_GLOBAL/ ← shared assets, HDRIs, materials (read-heavy)
├── BLENDER_APPS/ ← Blender install (do not edit)
│ └── blender-app/ ← extracted Blender 4.5.7 binary
└── CONFIG_MASTER/ ← shared startup files, preferences, keymaps
Rule of thumb Save everything you want synced inside ~/studio/PROJECTS/. Files saved anywhere else will not reach the VPS.
04 — Commands

Command reference

All commands are available in any terminal after running the bootstrap. Type them as-is — no extra arguments needed unless noted.

Command What it does Direction
studio-pull Pulls PROJECTS, LIBRARY_GLOBAL, and CONFIG_MASTER from VPS. Use this when starting a new session or when a teammate has uploaded new work. ← VPS
studio-pull-projects Pulls PROJECTS only — skips library and config. Faster. Use this when you know only project files have changed. ← VPS
studio-push Immediately pushes your entire PROJECTS folder to the VPS. Useful after finishing a major save you want teammates to access right away instead of waiting for the 5-min cron. → VPS
studio-push-folder Push a single named project folder. Takes one argument: the folder name inside PROJECTS. 
studio-push-folder my_scene
 → VPS
studio-sync Runs a full pull (all folders) followed by a push. Use when you want to both receive latest changes and immediately publish your own. ← VPS → VPS
studio-status Shows files that differ between your local PROJECTS and the VPS. Read-only — makes no changes. Use this to check before pulling if you're unsure what changed. check
studio-ls Lists all files currently in the VPS PROJECTS folder with sizes. Good for confirming a file was actually uploaded. read
studio-log Streams the live sync log. Shows cron syncs, errors, and timestamps. Press Ctrl+C to exit. log
studio-open Launches Blender 4.5.7 in the background. app
05 — Workflows

Daily workflows

Starting a session

  1. Open terminal in VNC.
  2. Pull the latest work from VPS: studio-pull
  3. Launch Blender: studio-open
  4. Work normally. Your saves auto-push every 5 minutes.

Fetching a specific file a teammate uploaded

  1. Ask your teammate to confirm the upload is done (or check with studio-ls).
  2. Run studio-pull-projects to bring it in quickly.
  3. Your file will appear in ~/studio/PROJECTS/.

Finishing a session and handing off to a teammate

  1. Save your blend file inside ~/studio/PROJECTS/your_folder/.
  2. Force-push immediately so they don't have to wait: studio-push
  3. Tell your teammate to run studio-pull-projects.

Uploading a packed .blend from your local machine to the VPS

If you have prepared a packed file offline and need to get it into the system:

  1. Copy your .blend file into the correct folder under ~/studio/PROJECTS/ on the VNC desktop (via the VNC file transfer panel or another terminal).
  2. Push it up immediately: studio-push-folder your_project_folder
  3. Confirm it landed: studio-ls
Avoid conflicts Do not have two people working on the same .blend file at the same time. Rclone is a last-write-wins system — whoever pushes last overwrites the other. Coordinate before working on the same file.
06 — Team Rules

Rules for the whole team

Always save inside ~/studio/PROJECTS/

Files saved anywhere else on the VNC desktop are local only and will be lost when the session resets. If Blender asks where to save, navigate to ~/studio/PROJECTS/.

Use a subfolder per project or per person

Don't dump files directly into the root of PROJECTS. Create a folder like PROJECTS/renders_john/ or PROJECTS/client_abc/ to keep things clean and avoid overwriting each other's work.

Communicate before pushing a major change

The cron job syncs every 5 minutes. If you're about to push something large (a fully packed scene, new library, etc.), give the team a heads up so they know to pull before opening those files.

Don't edit LIBRARY_GLOBAL or CONFIG_MASTER manually

These are shared by everyone. Changes there affect the whole team. Talk to the team lead before modifying anything in those folders.

Pack your blend files before uploading for handoff

If you're handing a file to someone else or archiving it, use Blender's File → External Data → Pack All Into .blend so all textures and assets travel with it. This prevents broken paths on the receiving end.

07 — Troubleshooting

Common issues

Commands not found after bootstrap

The aliases live in ~/.bashrc but only load for new shell sessions. Fix it by running:

source ~/.bashrc

rclone says "connection refused" or "ssh: handshake failed"

Your SSH key may have been reset or the session is stale. Re-run the bootstrap with the correct VPS_SSH_KEY_B64 to re-write the key and reconnect.

studio-status shows files that "differ" but you haven't changed them

This usually means timestamps differ without content changes — rclone uses checksums when it can, but SFTP falls back to size+time. It's safe to push — rclone will only transfer files that actually need updating.

Blender can't find textures after pulling a file

The file was likely not packed before upload. Ask the original author to re-pack and push, or manually redirect missing paths using Blender's File → External Data → Find Missing Files pointed at ~/studio/LIBRARY_GLOBAL/.

A file I pushed isn't showing on the VPS

# Check if it exists locally first
ls ~/studio/PROJECTS/your_folder/

# Then confirm VPS received it
studio-ls

# If it's still missing, force push manually
studio-push

The log shows errors

studio-log
# Press Ctrl+C to stop tailing

Look for lines starting with ERROR. Most errors are network timeouts — they will retry on the next cron cycle automatically.

08 — FAQ

Frequently asked questions

Do I need to push manually? Won't the cron do it?

The cron pushes every 5 minutes, so for normal working it handles everything. Manual studio-push is only for when you need a teammate to have your files right now — for example, handing off to the next person in a render pipeline.

What happens if I forget to pull before working?

Your work won't be lost, but you may overwrite a teammate's newer version when you push. Always run studio-pull-projects at the start of a session to stay safe.

Can I use Blender's own auto-save?

Yes. Blender's auto-save creates temp files in its own temp directory. Just make sure your real saves go into ~/studio/PROJECTS/ so they reach the VPS.

Will my work survive a VNC session reset?

Yes — as long as you saved inside ~/studio/PROJECTS/ and it had time to push (within 5 minutes). After a session reset, just run studio-pull-projects and your files will come back from the VPS.

How do I check what version of Blender I'm running?

~/studio/BLENDER_APPS/blender-app/blender --version

How do I add a new asset to LIBRARY_GLOBAL?

Talk to the team lead. Library changes affect everyone, so they go through a single gatekeeper to prevent conflicts.
